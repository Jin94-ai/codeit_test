# 협업 일지 작성 가이드

매일 작업 종료 전 10분 투자로 작성하세요.

---

## 파일명 규칙

```
YYYY-MM-DD_이름.md
```

**예시**:
- `2025-12-05_진석.md`
- `2025-12-06_민우.md`

---

## 템플릿

아래 내용을 복사해서 사용하세요:

```markdown
# 협업 일지 - YYYY-MM-DD (이름)

## 1. 오늘 한 일
-

## 2. 배운 점
-

## 3. 어려웠던 점
-

## 4. 내일 할 일
-

## 5. 팀원에게 공유하고 싶은 것
-
```

---

## 작성 팁

### 1. 오늘 한 일
- 구체적으로 작성 (예: "EDA 완료" → "나이별 알약 분포 분석 완료")
- 링크 추가 (PR, 노트북, 이슈 등)

### 2. 배운 점
- 기술적 학습 (예: "Albumentations의 BBoxSafeRandomCrop 사용법")
- 협업 학습 (예: "코드 리뷰를 통해 더 나은 변수명 선택법 배움")

### 3. 어려웠던 점
- 막힌 문제 명확히 기술
- 도움 요청 여부 표시

### 4. 내일 할 일
- 우선순위 표시 (1, 2, 3)
- 예상 소요 시간

### 5. 팀원에게 공유하고 싶은 것
- 유용한 자료/링크
- 발견한 버그나 이슈
- 개선 제안

---

## 예시

```markdown
# 협업 일지 - 2025-12-08 (민우)

## 1. 오늘 한 일
- EDA 노트북 작성 완료 ([notebooks/01_eda.ipynb](../../notebooks/01_eda.ipynb))
- 알약 크기 분포 분석: 평균 150x150px, 최소 50x50px
- 클래스별 데이터 불균형 확인: A클래스 45%, B클래스 30%, C클래스 25%

## 2. 배운 점
- Pandas의 groupby + agg 조합으로 복잡한 통계 한번에 계산
- Seaborn의 pairplot으로 다변량 관계 한눈에 파악

## 3. 어려웠던 점
- 바운딩 박스 좌표 형식이 YOLO 형식인지 COCO 형식인지 불명확
  → 수진님께 내일 확인 예정

## 4. 내일 할 일
1. (우선) 전처리 파이프라인 구현 시작 (4시간 예상)
2. 데이터 증강 전략 리서치 (2시간 예상)
3. 수진님과 모델 인풋 형식 논의

## 5. 팀원에게 공유하고 싶은 것
- Albumentations 공식 예제: https://github.com/albumentations-team/albumentations_examples
- 데이터 불균형 심하니 클래스 가중치 고려 필요할 듯
```
