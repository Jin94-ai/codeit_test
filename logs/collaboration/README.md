# 협업 일지 작성 가이드

매일 작업 종료 전 10-15분 투자로 작성하세요.

---

## 파일명 규칙

```
YYYY-MM-DD_이름.md
```

**예시**:
- `2025-12-05_진석.md`
- `2025-12-06_민우.md`

---

## 템플릿

아래 내용을 복사해서 사용하세요:

```markdown
# Daily 협업일지

### [1] 오늘 날짜 / 이름 / 팀명

- 날짜: YYYY-MM-DD
- 이름:
- 팀명: 코드잇 8팀

---

### [2] 오늘 맡은 역할 및 구체적인 작업 내용

> 오늘 당신이 맡았던 역할은 무엇이었고, 어떤 작업을 수행했나요?
>
> *(예: 모델 학습 파라미터 조정, 결측치 처리, 발표자료 구성 등)*

✍️ 답변:

(예시) 데이터 전처리 파트 담당. 범주형 변수 인코딩 및 이상치 확인 코드 작성 후 팀에게 공유함.

---

### [3] 오늘 작업 완료도 체크 (하나만 체크)

> 진척 상황을 정량적으로 표시하고, 간단한 근거도 작성하세요.

- [ ]  🔴 0% (시작 못함)
- [ ]  🟠 25% (시작은 했지만 진척 없음)
- [ ]  🟡 50% (진행 중, 절반 이하)
- [ ]  🔵 75% (거의 완료됨)
- [ ]  🟢 100% (완료 및 점검까지 완료)

📌 간단한 근거:

(예시) 75%: 주요 전처리 완료했지만 시각화 일부 남음

---

### [4] 오늘 협업 중 제안하거나 피드백한 내용이 있다면?

> 오늘 회의나 메시지에서 당신이 제안하거나 팀에 피드백한 내용은 무엇인가요?

✍️ 답변:

(예시) 모델 학습 전에 EDA를 먼저 해야 한다는 의견 제안. 기존 모델 성능 비교표 작성 아이디어 냄.

---

### [5] 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점은?

> EDA, 모델 실험 중 유의미한 점이나 오류가 있었다면 자유롭게 작성하세요.

✍️ 답변:

(예시) 'duration' 변수의 이상치가 모델 성능 저하에 영향을 주는 것으로 보임.

---

### [6] 일정 지연이나 협업 중 어려웠던 점이 있다면?

> 자기 업무 외에도 전체 일정이나 팀 내 협업에서 생긴 문제를 공유해 주세요.

✍️ 답변:

(예시) 발표자료 수정 중 겹치는 작업이 발생해 버전 충돌 있었음 → 구글 슬라이드로 일원화함.

---

### [7] 오늘 발표 준비나 커뮤니케이션에서 기여한 부분은?

> 슬라이드 제작, 발표 연습, 질문 정리 등 발표와 관련된 활동을 썼다면 기록하세요.

✍️ 답변:

(예시) 발표용 그래프 수정 및 정렬. 질문 예상 리스트 작성해서 팀 공유.

---

### [8] 내일 목표 / 할 일

> 구체적인 개인 업무나 팀 목표 기반 계획을 간단히 적어주세요.

✍️ 답변:

(예시) 모델 성능 비교 실험 3개 수행 후 결과 요약, 발표자료 3p 정리

---
```

## 작성 예시

### 2025-12-05_민우.md

```markdown
# Daily 협업일지

### [1] 오늘 날짜 / 이름 / 팀명

- 날짜: 2025-12-05
- 이름: 김민우
- 팀명: 코드잇 8팀

---

### [2] 오늘 맡은 역할 및 구체적인 작업 내용

✍️ 답변:

Data Engineer 역할. EDA 노트북 작성:
- 데이터 로드 및 기본 통계 확인
- 알약 클래스별 분포 분석
- 이미지 크기 및 바운딩 박스 분포 시각화
- notebooks/01_eda.ipynb 작성 완료

---

### [3] 오늘 작업 완료도 체크

- [x]  🔵 75% (거의 완료됨)

📌 간단한 근거:

EDA 주요 분석 완료. 데이터 증강 전략 수립 부분만 남음 (내일 완료 예정)

---

### [4] 오늘 협업 중 제안하거나 피드백한 내용이 있다면?

✍️ 답변:

- 데이터 클래스 불균형 문제 발견 → 가중치 또는 오버샘플링 제안
- 작은 알약(50px 이하) 검출 어려움 → 이미지 크기 증가 고려 제안

---

### [5] 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점은?

✍️ 답변:

- 알약 크기 분포: 평균 150x150px, 최소 50x50px
- 클래스 불균형: A클래스 45%, B클래스 30%, C클래스 25%
- 겹친 알약 케이스 약 15% → 모델 선택 시 고려 필요

---

### [6] 일정 지연이나 협업 중 어려웠던 점이 있다면?

✍️ 답변:

- 바운딩 박스 좌표 형식(YOLO vs COCO) 불명확 → 보윤님께 확인 예정
- Jupyter Notebook 환경 설정에 시간 소요 (30분)

---

### [7] 오늘 발표 준비나 커뮤니케이션에서 기여한 부분은?

✍️ 답변:

- EDA 결과 그래프 5개 생성 (발표 자료용)
- 데이터 분석 결과 팀 Discord에 공유

---

### [8] 내일 목표 / 할 일

✍️ 답변:

1. (우선) 데이터 증강 전략 수립 및 실험 (3시간)
2. 전처리 파이프라인 설계 시작 (2시간)
3. 보윤님과 모델 인풋 형식 논의

---
```

## 작성 팁

1. **구체적으로 작성**: "코딩함" → "YOLOv8 학습 코드 작성 (train.py)"
2. **수치 활용**: "많이 했음" → "3개 실험 완료, 2개 진행 중"
3. **솔직하게**: 어려웠던 점, 지연 사항도 공유
4. **내일 계획 명확히**: 우선순위와 예상 시간 포함

---
